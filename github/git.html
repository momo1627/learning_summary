version control

commit version
git reset --hard version ID
each commit will generate a version id; use git reset to go back or forward a committed version
git reset HEAD file
reset also can return version from added content to last commit
version log
git log/git relog
check commit log and version ID
reverse changes
git checkout -- file
reverse changes back to last git add or git commit
reverse file to last commit version

origin repo

push local repo and link to an origin repo
create an origin repo without readme
git remote add origin git@github.com:username/git
git push -u origin master
clone an origin repo to local
create an origin repo with initial readme
select local workplace
git clone git@github.com:username/xx.git

branch

create and merge/delete branch
git checkout -b name = git branch name/git checkout name
git merge dev = merge dev to master
git branch -d name : delete branch
fix conflicks
git merge --no-ff -m 'message' name
merge but not delete branch
